## 1. 引言

### 1.1 编写目的

本文档为「估值监测工具」MVP 阶段的需求基线，用于：
- 锚定后续系统设计、技术选型与实现的功能范围
- 在开发开始前显式固化业务规则（百分位窗口、阈值表、调度时刻、回溯策略等），避免设计与实现阶段反复返工
- 作为验收依据，每一项功能性需求与关键决策均可对照检查

### 1.2 项目背景

作为个人投资者，作者长期关注 A 股 / 港股 / 美股各类指数的估值水平，但现有工具的痛点突出：

- **数据散落**：A 股估值数据多看「中证指数公司」「理杏仁」，港股看恒生指数公司或第三方，美股需 yfinance / Multpl 等，跨市场比对成本高
- **指标口径不一**：不同平台 PE-TTM、股息率口径不一，历史百分位窗口（5 年 / 10 年 / 全历史）也不统一
- **无个性化阈值**：通用平台的「低估/高估」是固定档位，无法按指数风格（宽基 vs 行业 vs 红利）个性化
- **缺少定投联动**：估值数据与定投计划脱节，需手动判断「本月是否加倍」
- **A/H/美三地割裂**：单一工具难以一屏总览三地估值水位

本工具旨在以**个人自用、本地部署、每日批处理**的形式解决以上痛点。

### 1.3 术语表

| 术语 | 定义 |
|---|---|
| 指数（Index） | 一组成分股按特定规则加权计算的指数化产品（如沪深 300、恒生指数、标普 500） |
| 宽基指数 / 行业指数 / 主题指数 | 分类口径：宽基覆盖整个市场（沪深 300）；行业按 GICS/申万等分类（消费、医药）；主题按概念（红利、低波、ESG） |
| PE-TTM | 滚动 12 个月市盈率 = 当前指数点位 / 成分股加权 TTM 净利润；本系统优先采用指数公司官方披露值，缺失时按成分股加权反算 |
| PB | 市净率 = 指数点位 / 成分股加权净资产 |
| 股息率 | 近 12 个月成分股加权分红 / 当前指数点位 |
| 历史百分位 | 当前指标值在过去 N 年序列中的排序位置（0%–100%）；本系统的信号与温度均以**近 10 年**窗口为锚 |
| 估值温度 | 0–100 的归一化温度值，定义为 **PE-TTM 历史百分位 × 100**（一一映射，下同） |
| 估值档位 | 由温度落入的区间映射出的标签：极度低估 / 低估 / 合理 / 高估 / 极度高估 |
| 定投（DCA） | 定期定额买入，本系统额外支持依据估值温度调整投入比例 |
| 复权 | 对历史价格按分红/配股事件做向前或向后调整以保持连续性 |
| ETF / 场外基金 | 跟踪指数的两类公募产品；ETF 为场内交易（按市价），场外基金按净值申赎 |

## 2. 总体描述

### 2.1 产品愿景

> **一屏看清全球主要指数的估值水位，并给出可执行的定投与买卖时机提示。**

### 2.2 用户画像

| 项 | 描述 |
|---|---|
| 角色 | 个人投资者（系统唯一用户） |
| 投资风格 | 长期主义 + 指数化投资，使用宽基与行业 ETF 做核心 + 卫星配置 |
| 技术能力 | 能够本地部署 Docker / Python 应用，熟悉浏览器操作 |
| 主要任务 | 每日浏览估值总览；为高估指数减少定投，为低估指数增加定投；偶尔做轻量回测 |
| **非目标** | 不做主动交易、不做日内、不做杠杆与衍生品 |

### 2.3 运行环境

- **客户端**：现代桌面浏览器（Chrome 100+、Safari 15+、Edge 100+、Firefox 100+）；移动端响应式仅做布局适配
- **服务端**：本机或个人 VPS（Linux x86_64，4 GB 内存以上）
- **网络**：能访问 AkShare / yfinance / Tushare 等数据源
- **部署形态**：默认 `127.0.0.1:8000`，**仅本机回环监听**

### 2.4 约束与假设

| 类别 | 内容 |
|---|---|
| 范围约束 | 不做用户体系（单用户）、不做实时行情、不做仓位管理 |
| 数据假设 | 数据源可用；同一指数在不同数据源间口径差异在可接受范围 |
| 跨市场比较 | **仅在分位数（去量纲）层面进行**——原币种 PE/PB 绝对值不互比；UI 必须标注币种 |
| 汇率 | 不纳入估值口径与展示；ETF 净值按其本币计算 |
| 法律合规 | 个人自用研究工具，不构成投资建议，不分发、不收费 |

## 3. 功能性需求

### FR-1 指数与基金映射管理

**目标**：在系统内维护一份可用指数清单及其对应的可投资基金/ETF。

**输入**：
- 预置 YAML 文件 `data/index_universe.yaml`（首次部署导入）
- 用户在 Web 端勾选自选列表

**输出**：
- 内置指数池（首版约 30–50 个指数，覆盖三地主流宽基 + 行业 + 主题）
- 用户自选列表（持久化到 `Watchlist`）

**业务规则**：
1. 内置指数池由开发者手工维护 YAML，**用户不能新增指数**（MVP 限制，避免数据源覆盖不到）
2. 用户仅能从池中勾选/取消自选
3. 每条指数记录至少含：`code`、`name`、`market`（A/HK/US）、`category`（宽基/行业/主题）、`industry_raw`（原市场分类原值，可空）、`data_source`、`history_start_date`
4. 每条 `Fund` 记录至少含：`code`、`name`、`type`（ETF/场外）、`tracks_index_id`、`fee_rate`、`tracking_error_note`、`market`
5. 一个指数可对应多个基金（如沪深 300 对应华泰柏瑞 510300、易方达 510310、华夏 510330 等）

**示例预置条目**（YAML 片段）：
```yaml
- code: "000300.SH"
  name: "沪深300"
  market: "A"
  category: "宽基"
  industry_raw: null
  data_source: "akshare"
  history_start_date: "2005-04-08"
  funds:
    - { code: "510300.SH", name: "华泰柏瑞沪深300ETF", type: "ETF", fee_rate: 0.0015 }
    - { code: "110020", name: "易方达沪深300ETF联接A", type: "OPEN_FUND", fee_rate: 0.005 }
```

### FR-2 数据采集与入库

**目标**：每日收盘后从数据源拉取指数行情与估值原始数据，按市场调度，容错落库。

#### 输入
- `index_universe.yaml` 中的指数清单
- 调度时间（D3 时刻表）
- 数据源凭证（环境变量）

#### 输出
- `IndexQuote` 表新增/覆盖记录
- `DataAudit` 表的回溯审计日志
- 数据源健康状态（最近一次成功 / 失败时间）

#### 调度时刻表（D3）

| 市场 | 触发时间（UTC+8） | 备注 |
|---|---|---|
| A 股 | 16:30 | 收盘 15:00 + 30 分钟缓冲，等待估值数据更新 |
| 港股 | 17:30 | 收盘 16:00 + 90 分钟缓冲 |
| 美股 | 次日 07:00 | 美东 19:00 左右，盘后数据齐 |

- 每个市场使用各自 `TradingCalendar`，非交易日跳过
- 同一调度任务内**仅处理该市场指数**，避免跨时区数据混淆

#### 数据回溯策略（D2）

1. 每次拉取**当日 + 前 30 个交易日**的 `IndexQuote`，覆盖已入库记录（处理财报回溯、分红除权等历史修正）
2. 拉取完成后，对**最近 30 个交易日**重算 `Valuation` 派生分位（早于 30 天的派生分位锁定不动）
3. 每次覆盖动作必须写 `DataAudit` 日志：`date`, `index_code`, `field`, `old_value`, `new_value`, `source`, `audit_time`

#### 容错策略

- 单一数据源拉取失败 → 跳过该市场当日 → 记录错误日志 → **次日全量补拉前 30 天**（不做断点续传状态机）
- 字段缺失但非关键（如 ROE）→ 记录为 NULL，UI 端友好回退（"—"）
- 关键字段缺失（PE-TTM）→ 不写入该日记录，告警

#### 完整性校验

每次入库后执行：
- 当日是否所有市场已拉取
- 当日是否每个指数都有 PE-TTM 值（缺失则告警）
- 当日指数点位与昨日相比变动是否超过阈值 ±15%（异常告警）

### FR-3 估值计算引擎

**目标**：基于 `IndexQuote` 计算 PE/PB/股息率历史百分位、估值温度、档位与辅助指标。

#### 计算口径声明

| 指标 | 口径 |
|---|---|
| PE-TTM | 优先使用聚合源（乐咕乐股）披露的指数 PE-TTM 历史序列；缺失时用成分股加权 TTM 净利润反算（仅 A 股可稳定支持） |
| PB | 聚合源披露的指数 PB 历史序列；缺失时按成分股加权净资产计算 |
| 股息率 | 近 12 个月成分股加权现金分红 / 当前指数点位 |
| ROE | 成分股加权 ROE（**仅 A 股稳定可用**，港美股为 best-effort，缺失时不展示该字段） |
| 盈利增长 | 近 3 年成分股加权净利润 CAGR（**仅 A 股稳定可用**） |

##### A 股数据源接口选型说明（M1 验证后更正）

| 用途 | 接口（akshare） | 备注 |
|---|---|---|
| **PE-TTM 历史 + 指数点位** | `stock_index_pe_lg(symbol=中文名)` | 乐咕乐股聚合，含 5000+ 行（2005 起），支持 沪深300/中证500/800/1000/上证50 等 |
| **PB 历史** | `stock_index_pb_lg(symbol=中文名)` | 同上，5000+ 行历史 |
| 行情 OHLC + 成交量（M2 起） | `index_zh_a_hist(symbol=数字代码, period='daily')` | 东方财富，用于 MA50/MA200 偏离 |
| 交易日历 | `tool_trade_date_hist_sina()` | 新浪 |
| 股息率历史 | **当前数据源缺口**（M2 待解决） | `stock_zh_index_value_csindex` 只给最近 20 个交易日快照，不适合分位计算 |

> **重要更正**：早期假设 `stock_zh_index_value_csindex` 提供历史估值，M1 实测仅返回最近 20 行快照（且无指数 close、无 PB）。已切换到乐咕乐股聚合接口。

#### 百分位窗口（D1）

- **默认窗口：近 10 年**
- UI 可切换：「近 5 年 / 近 10 年 / 全历史」（仅用于浏览，**不触发信号**，避免规则随窗口漂移）
- 信号与温度计算固定锚定**近 10 年**

#### 估值档位与温度映射表（D1）

| 档位 | PE 百分位 | 温度 | 含义 | 默认信号 |
|---|---|---|---|---|
| 极度低估 | < 10% | 0–10 | 强买入 | 触发"强买入"信号 |
| 低估 | 10–30% | 10–30 | 买入 | 触发"买入"信号 |
| 合理 | 30–70% | 30–70 | 持有 | 不触发 |
| 高估 | 70–90% | 70–90 | 减持 | 触发"减持"信号 |
| 极度高估 | > 90% | 90–100 | 强减持 | 触发"强减持"信号 |

- **温度公式**：`温度 = PE 百分位 × 100`（一一映射，不引入额外曲线函数）
- **温度仅以 PE-TTM 为锚**；PB、股息率独立计算分位用于辅助展示，不参与温度
- 用户可对单一指数通过 `ThresholdOverride` 覆盖任一边界（参见 FR-5）

#### 历史数据不足的处理

| 可用历史长度 | 处理 |
|---|---|
| ≥ 10 年 | 正常使用 10 年窗口 |
| 5–10 年 | 用可用最长窗口（如 7 年）计算分位，UI 加角标"窗口=N年"提示 |
| < 5 年 | UI 置灰显示"分位不可用"，**不参与信号生成**，**不允许绑定定投计划** |

#### 辅助指标（D4，MVP 仅以下两项）

| 指标 | 适用市场 | 口径 |
|---|---|---|
| 50/200 日均线偏离度 | A/H/US 通用 | (当前价 − MA50) / MA50，(当前价 − MA200) / MA200 |
| 北向资金累计净流入 60 日分位 | **仅 A 股** | 过去 60 个交易日累计北向净流入在过去 5 年同窗口序列中的分位数 |

成交量分位、波动率、换手率等**列入未来扩展**，MVP 不实现。

#### 计算示例（强制示例：沪深 300）

**步骤**：
1. 从 `IndexQuote` 中拉取沪深 300 过去 10 年（约 2430 个交易日）的 `pe_ttm` 序列
2. 将该序列升序排序
3. 找到**当日 PE-TTM** 在排序序列中的位置 `rank`（含相同值，取均值法）
4. 百分位 = `rank / 总数`
5. 温度 = 百分位 × 100
6. 按上表映射档位

**样例数据片段**（说明用，非真实历史数据）：

| date | close | pe_ttm | pe_percentile_10y | temperature | tier |
|---|---|---|---|---|---|
| 2026-05-08 | 3520.45 | 12.85 | 0.235 | 23.5 | 低估 |
| 2026-05-09 | 3502.10 | 12.78 | 0.227 | 22.7 | 低估 |
| 2026-05-10 | 3495.66 | 12.75 | 0.224 | 22.4 | 低估 |
| 2026-05-11 | 3510.32 | 12.81 | 0.231 | 23.1 | 低估 |
| 2026-05-12 | 3528.90 | 12.88 | 0.239 | 23.9 | 低估 |

> 字段命名直接对应 `Valuation` 表，便于后续建表。

### FR-4 监控仪表盘（Web UI）

**目标**：以视觉方式呈现估值水位，支持总览、详情、自选、信号、定投五个主页面。

#### 总览页

- **三栏布局**（D5）：A 股 / 港股 / 美股，每栏独立
- 每栏内部按用户标签或类别分组（宽基 / 行业 / 主题）
- 主视图：**估值热力图**（每个指数为一个色块，颜色按温度档位映射，绿→红渐变）
- 副视图：可切换为表格视图（列：指数名、温度、PE、PE 分位、PB 分位、股息率、对应 ETF 涨跌）
- 表格默认按温度升序，便于发现低估
- 标注币种（CNY / HKD / USD）

#### 详情页

- 顶部：指数元信息卡（名称、市场、类别、对应 ETF/基金列表含费率）
- 主图：双轴折线
  - 主轴：指数收盘价（10 年）
  - 副轴：PE-TTM 折线，叠加百分位带（10/30/70/90 四条水平参考线，对应档位边界）
  - 标注当前点位
- 副图：PB-TTM 同样形式；股息率折线
- 辅助卡：50/200 日均线偏离度、（A 股）北向资金 60 日分位
- 操作：加入/移出自选、设置个性化阈值

#### 自选页

- 仅展示用户 Watchlist 中的指数，按当前温度排序
- 突出显示落入"低估及以下"或"高估及以上"档位的指数

#### 信号页

- 今日信号列表（按市场分组）
- 历史信号检索（按日期、指数、档位筛选）
- 每条信号含：触发时间、指数、温度、档位、对应 ETF/基金推荐链接

#### 定投页

参见 FR-5。

### FR-5 估值信号与定投提醒

**目标**：根据估值档位生成信号；按定投计划与温度联动给出"加倍/正常/减半/暂停"建议。

#### 信号生成

- 每日批处理后，对每个指数评估：
  - 若进入「极度低估 / 低估 / 高估 / 极度高估」档位 → 生成对应方向信号
  - 信号**全局产生**，但在 UI 中仅展示给"该指数在 Watchlist 或 DCAPlan 中"的用户视图
- 个性化阈值：用户可通过 `ThresholdOverride` 覆盖默认档位边界，覆盖后信号判断使用覆盖后的边界

#### ThresholdOverride 字段粒度

```
ThresholdOverride {
  id: int
  index_id: FK
  boundaries_json: {
    "extreme_low_upper": 0.10,   # 极度低估上限
    "low_upper":         0.30,   # 低估上限
    "high_lower":        0.70,   # 高估下限
    "extreme_high_lower":0.90,   # 极度高估下限
  }
  // 未指定的边界回退到默认值
  updated_at: timestamp
}
```

#### 定投计划（DCAPlan）

| 字段 | 说明 |
|---|---|
| `index_id` | **必填**，每个定投计划必须绑定到一个指数（即使买入的是基金，也追溯到其跟踪指数） |
| `fund_id` | 实际买入的基金/ETF（可选，仅记录用） |
| `amount` | 单次定投基础金额（本币） |
| `frequency` | 周期（每周/每两周/每月） |
| `day_of_period` | 触发日（每周几 / 每月几日） |
| `start_date` | 起始日 |
| `enabled` | 启用开关 |

#### 调整规则（D6，与档位强联动 — 方案 A）

| 当前档位 | 本期建议 | 说明 |
|---|---|---|
| 极度低估 / 低估 | **加倍**（2× 基础金额） | 触发更激进买入 |
| 合理 | **正常**（1× 基础金额） | 维持节奏 |
| 高估 | **减半**（0.5× 基础金额） | 降低买入强度 |
| 极度高估 | **暂停**（0×） | 不买入，等待回落 |

- **联动**：若用户为该指数覆盖了 ThresholdOverride 边界，调整规则随覆盖边界**同步移动**——"加倍"始终对应"低估及以下档位"，无论用户把"低估上限"调成 30% 还是 15%
- UI 在定投提醒卡片底部以小字注明：「档位边界来源：默认 / 自定义（10% / 15% / 70% / 90%）」

#### 定投日非交易日回退

若 `day_of_period` 落在非交易日（周末、市场假日、跨年休市），**顺延至该市场下一个交易日**；提醒在新定投日前 1 个交易日 23:00 前生成。

| 场景 | 用户配置 | 实际定投日 | 提醒生成日 |
|---|---|---|---|
| 周末 | 每月 10 日（A 股，2026 年 5 月 10 日为周日） | 2026-05-11 周一 | 2026-05-08 周五批处理后 |
| 市场假日 | 每月 1 日（A 股，2026 年 5 月 1 日为劳动节假期） | 2026-05-06 周二（劳动节后首个交易日） | 2026-05-05 批处理后 |
| 跨年 | 每月 30 日（美股，2026 年 12 月 30 日休市叠加跨年） | 2027-01-04 | 2026-12-31 批处理后 |

#### 端到端示例（强制示例：中证红利定投）

**用户配置**：
- 指数：中证红利（000922.CSI）
- 基金：富国中证红利指数增强（100032）
- 金额：¥2000
- 频率：每月 10 日

**触发链路**：
1. 2026-05-12 批处理后，发现 5 月 11 日（前一交易日）该指数温度 = 18.3 → 落入"低估"档位
2. 由于用户未覆盖默认阈值，使用默认边界（10/30/70/90）
3. 本期定投建议：**加倍 → ¥4000**
4. 系统在「定投页」生成提醒卡片：

### FR-6 数据导出与回测

#### 数据导出
- 任一指数的 `IndexQuote` + `Valuation` 历史数据可导出 CSV
- 字段：日期、收盘、PE、PB、股息率、PE 百分位、PB 百分位、温度、档位

#### 轻量回测（D9）

**输入**：
- 单一指数
- 单一阈值策略：`PE 百分位 < X → 全仓买入`，`PE 百分位 > Y → 全仓卖出`
- 回测起止日期（默认起 = 数据起始日，止 = 昨日）

**输出（仅 2 个指标）**：
- 年化收益率
- 最大回撤

**简化假设**：
- 全仓单笔操作，不分批
- **不含**手续费、滑点、税费、分红再投资
- 信号当日收盘买入/卖出
- 假设资金充足

**UI**：
- 单页表单（指数选择、X、Y、起止日期）
- 点击「运行」展示结果，**不保存策略**

## 6. 数据模型

### 6.1 实体清单与关键字段

| 实体 | 关键字段 |
|---|---|
| `Market` | `id`, `code` (A/HK/US), `name`, `currency`, `tz` |
| `TradingCalendar` | `id`, `market_id`, `date`, `is_open` |
| `Index` | `id`, `code`, `name`, `market_id`, `category`, `industry_raw`, `data_source`, `history_start_date` |
| `Fund` | `id`, `code`, `name`, `type`, `tracks_index_id`, `fee_rate`, `tracking_error_note`, `market_id` |
| `IndexQuote` | `id`, `index_id`, `date`, `close`, `pe_ttm`, `pb`, `dividend_yield`, `roe`, `earnings_growth_3y` |
| `Valuation` | `id`, `index_id`, `date`, `window` (5y/10y/all), `pe_percentile`, `pb_percentile`, `dy_percentile`, `temperature`, `tier` |
| `Watchlist` | `id`, `index_id`, `tag`, `added_at` |
| `Signal` | `id`, `index_id`, `date`, `direction` (strong_buy/buy/sell/strong_sell), `temperature`, `tier`, `generated_at` |
| `DCAPlan` | `id`, `index_id`, `fund_id`, `amount`, `frequency`, `day_of_period`, `start_date`, `enabled` |
| `DCAExecution` | `id`, `plan_id`, `scheduled_date`, `actual_date`, `adjusted_amount`, `tier_at_decision`, `status` (pending/done/skipped) |
| `ThresholdOverride` | `id`, `index_id`, `boundaries_json`, `updated_at` |
| `DataAudit` | `id`, `table`, `record_key`, `field`, `old_value`, `new_value`, `source`, `audit_time` |
| `UserPreference` | `id`, `key`, `value_json`（如默认百分位窗口、UI 主题、热力图配色等） |

### 6.2 关系说明（文字版 ER）

- `Market 1—* Index`：一个市场含多个指数；指数总属于一个市场
- `Market 1—* TradingCalendar`：每个市场有独立的交易日历
- `Index 1—* Fund`：一个指数可对应多个跟踪基金
- `Index 1—* IndexQuote`：日频原始数据，主键 `(index_id, date)`
- `Index 1—* Valuation`，且对每个 `(index_id, date)` 有 N 条 Valuation（窗口 5y/10y/all），即 **IndexQuote : Valuation = 1 : N**
- `Index 1—1 ThresholdOverride`（用户最多对每个指数覆盖一次，可空）
- `Index 1—* Watchlist`（一个指数可在多个 tag 下）
- `Signal *—1 Index`：信号针对指数全局产生，由 UI 过滤为"用户感兴趣的"（即 Watchlist 或 DCAPlan 涉及的指数）
- `DCAPlan *—1 Index`，`DCAPlan *—1 Fund`（fund 可空）
- `DCAPlan 1—* DCAExecution`：每次定投到期产生一条执行记录
- `DataAudit` 不与业务实体形成外键，仅按字符串主键引用，便于跨表审计

### 6.3 索引建议

- `IndexQuote (index_id, date)` 唯一索引
- `Valuation (index_id, date, window)` 唯一索引
- `Signal (date, index_id)` 联合索引（按日期反向查询）
- `TradingCalendar (market_id, date)` 唯一索引

### 6.4 PG 迁移预留（D7 附录）

未来如迁移 PostgreSQL：
- SQLite `REAL` → PG `NUMERIC(18, 6)`（避免浮点漂移影响分位计算）
- SQLite `INTEGER` 自增 → PG `BIGSERIAL`
- JSON 字段：SQLite TEXT → PG `JSONB`
- 全文检索（如指数名搜索）：SQLite FTS5 → PG `tsvector` + GIN 索引

## 9. 风险与缓解

| 风险 | 影响 | 缓解 |
|---|---|---|
| AkShare/yfinance 接口偶发变更 | 数据采集失败 | 数据源适配器抽象 + 多源回退 + 容错跳过 + 次日补拉 |
| 港美股指数级 PE-TTM 口径差异 | 跨市场不可比 | 仅在分位数层面比较；UI 标注口径来源 |
| 港美股 ROE / 盈利增长数据缺口 | 字段空值 | 标记 best-effort，UI 空值友好回退 |
| 财报回溯导致历史 PE 变动 | 历史分位漂移 | 每日回溯近 30 天 + 锁定更早派生分位 + 审计日志 |
| 历史数据起始时间不足 10 年的指数 | 分位失真 | < 5 年置灰禁用；5–10 年用可用最长窗口并加角标。**判定窗口须以 `index_quote.MIN(date)` 实际覆盖为准**，不可只看 `index_meta.history_start_date`（M1 实现使用后者是已知缺陷，M2 修复） |
| 三地交易日历与时区调度复杂 | 调度错乱 | TradingCalendar 实体 + 三地独立调度 |
| 分红口径（近 12 月 vs 财年） | 股息率不一致 | SRS 中已声明用近 12 月分红口径 |
| 用户阈值覆盖与定投规则联动复杂 | UX 困惑 | 联动方案 A + UI 明示边界来源 |
| SQLite 单文件并发写入 | 多任务竞争 | 调度三地分时段错峰；写操作集中在单一调度线程 |

| **D18** | 架构：**Stock / Fund 各自独立实体表**（不复用 IndexMeta）；复用 `app/valuation/` 纯函数库（percentile/temperature/tier）但不复用表 | FR-7 §11.2.1 / FR-8 §11.3.2 |

### 11.2 FR-7 个股估值监测

**目标**：把现有指数估值监测能力扩展到 A 股个股；用户可手动添加自选个股，系统自动拉取上市以来全历史 PE/PB/PS/股息率，按行业默认估值锚计算温度与档位。

**输入**：
- 用户在 Web 端添加股票代码（如 `600519.SH`）
- 数据源：akshare `stock_a_indicator_lg`（个股 PE/PB/PS 历史）+ akshare `stock_individual_info_em`（行业 / 上市日 / 总股本）+ Tushare `daily_basic`（可选兜底）

**输出**：
- `Stock` 实体（自选池）
- `StockQuote` 日频行情 + 估值原始字段
- `StockValuation` 派生分位 + 温度 + 档位（按个股锚）
- 自选个股总览 + 个股详情页

#### 11.2.1 数据模型扩展

| 实体 | 关键字段 | 备注 |
|---|---|---|
| `Stock` | `id`, `code` (`600519.SH`), `name`, `market_id` (固定 A), `sw_industry_1`, `sw_industry_2`, `sw_industry_3`, `listing_date`, `valuation_anchor` (`PE`/`PB`/`PS`/`PE_REVERSE`/`DIV_YIELD`), `status` (`active`/`delisted`/`suspended`), `enabled`, `added_at` | 行业取首次入库快照锁定；公司主业转型 / 申万分类调整时**不自动更新**，需手动 review |
| `StockQuote` | `id`, `stock_id`, `date`, `close`, `pe_ttm`, `pb`, `ps_ttm`, `dividend_yield`, `roe_ttm`, `total_share`, `market_cap`, `source`, `created_at` | 字段结构与 `IndexQuote` 类比但**不含** `ma50/ma200/northbound_60d_pct`（R12 MVP 不接技术面） |
| `StockValuation` | `id`, `stock_id`, `date`, `window`, `source`, `anchor` (`PE`/`PB`/`PS`/`PE_REVERSE`/`DIV_YIELD`), `pe_percentile`, `pb_percentile`, `ps_percentile`, `dy_percentile`, `temperature`, `tier`, `computed_at` | `anchor` 字段记录**实际用于算温度的锚**；窗口含 5y/10y/all |

- `Stock(code)` 唯一
- `StockQuote(stock_id, date)` 唯一

#### 11.2.3 温度公式（5 种锚）

| 锚 | 温度公式 | 含义 |
|---|---|---|
| `PE` | `pe_percentile × 100` | 同指数（沿用 D1）：高 PE 百分位 = 高估值 |
| `PB` | `pb_percentile × 100` | 高 PB 百分位 = 高估值；适用于银行 / 房地产 |
| `PS` | `ps_percentile × 100` | 高 PS 百分位 = 高估值；适用于早期 SaaS / 互联网 |
| `PE_REVERSE` | `(1 − pe_percentile) × 100` | **PE 倒置**：低 PE = 顶部（钢铁 / 煤炭 / 化工等周期股） |
| `DIV_YIELD` | `(1 − dy_percentile) × 100` | **股息率倒置**：股息率低 = 高估（公用事业 / 交通运输） |

阈值表沿用 D1（0–10 极度低估 / 10–30 低估 / 30–70 合理 / 70–90 高估 / 90–100 极度高估），不为个股引入新档位。

**实施约束**：
- 数据点 < 250 时不计算温度（沿用 R7 `MIN_DATAPOINTS_FOR_PERCENTILE`）
- 个股若所选锚字段缺失（如刚上市股票 PS 历史不足），UI 显示"快照模式"角标，不强制温度

### 11.3 FR-8 基金估值监测

**目标**：监测自选基金的估值水位。三类基金对应不同估值方法（D16）：

| 基金类型 | 估值方法 | 温度来源 |
|---|---|---|
| 场内 ETF（如 510300） | **挂跟踪指数温度** | 复用 `Valuation.temperature` |
| 场外指数基金（如 110020） | **挂跟踪指数温度** | 同上 |
| 场外主动基金（如 005827） | **NAV 历史百分位**（"与自己比"） | 单独的 `FundValuation` |

#### 11.3.1 三类基金对应估值方法（D16）

- **场内 ETF / 场外指数基金**：`Fund.tracks_index_id` 非空 → 详情页直接展示跟踪指数的温度。在 ETF 自身字段（净值折溢价、年化跟踪误差）中加显示但不改温度。
- **场外主动基金**：`Fund.tracks_index_id` 为空 → 计算 `FundValuation` 用 NAV 历史百分位。
  - 公式：`temperature = nav_percentile_5y × 100`（窗口固定 5y，不足时用可用最长）
  - 含义：高百分位 = NAV 处于近 5 年高位（"与自己比贵"）
  - **限制**：仅反映"基金净值与自身历史"，不反映"基金持仓估值水位"；UI 须明示

#### 11.3.2 数据模型扩展

| 实体 | 关键字段 | 备注 |
|---|---|---|
| `Fund` 扩展 | 增加 `fund_type` (`ETF`/`INDEX_FUND`/`ACTIVE_FUND`), `nav_source` (`tiantian`/`akshare_fund`), `enabled_for_monitor` | 现有 Fund 表（M1 时已建）补充字段；旧记录 fund_type 默认 'ETF' |
| `FundNAV` | `id`, `fund_id`, `date`, `nav` (单位净值), `accumulated_nav` (累计净值), `created_at` | 仅场外基金；ETF 走 IndexQuote 路径 |
| `FundValuation` | `id`, `fund_id`, `date`, `window` (5y/all), `nav_percentile`, `temperature`, `tier`, `computed_at` | 仅主动基金；档位沿用 D1 |

唯一约束：
- `FundNAV(fund_id, date)` 唯一
- `FundValuation(fund_id, date, window)` 唯一

| 个股 LG `stock_a_indicator_lg` 接口对小盘股 / 新股 / ST 股不稳定 | 部分股票拉不到 PE 历史 | Tushare `daily_basic` 兜底；UI 标注"数据源 N/A"友好降级 |
| 行业分类滞后于公司主业转型（如某计算机公司转 AI 半导体） | 估值锚不再合适 | StockOverride 允许用户手动改锚；MVP 不做自动跟踪 |
| 申万一级 → 锚 映射是开发者主观决定 | 用户认知与默认锚不符 | UI 详情页显示"锚来源：行业默认 / 用户覆盖"；锚切换下拉给所有 5 种锚选项 |
| 个股退市 / 长期停牌 | 历史 PE 失真，温度无意义 | `Stock.status` 标记；UI 灰显并隐藏温度 |
| 主动基金"NAV 与自己比"的统计意义有限（基金经理换、风格漂移） | 温度可能误导 | UI 醒目 ⚠ 注明"仅反映 NAV 自身历史，不反映持仓水位"；不接入信号 |
| 财报披露日 PE 跳变（D17 决定不特殊处理） | 个股温度可能短期失真 | 沿用 R11 DAILY_JUMP 告警；用户可标记已核对 |
| 个股数据量较大（50 只 × 6000 天 = 30 万行 StockQuote） | SQLite WAL 性能 | 与现有 IndexQuote (24 只 × 2500 = 6 万行) 相比可接受；如有性能问题 D7 PG 迁移路径已预留 |
